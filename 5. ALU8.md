# ALU8 — Commands in Order: RTL → GDS → KLayout → Magic

A single, ordered command reference for taking the 8-bit ALU (`alu8`) design from RTL all the way through to viewing the final layout in both **KLayout** and **Magic**. No explanations of *why* here — just the exact sequence, in order. For the reasoning behind each step, see the other docs in this repo.

---

## 0. RTL — `src/alu8.v`

```verilog
module alu8 (
    input  wire [7:0] a,
    input  wire [7:0] b,
    input  wire [2:0] sel,
    output reg  [7:0] result,
    output reg        carry_out
);

always @(*) begin
    carry_out = 1'b0;
    case (sel)
        3'b000: result = a + b;                       // ADD
        3'b001: {carry_out, result} = a - b;           // SUB
        3'b010: result = a & b;                        // AND
        3'b011: result = a | b;                        // OR
        3'b100: result = a ^ b;                         // XOR
        3'b101: result = ~a;                            // NOT
        3'b110: result = a << 1;                        // Logical shift left
        3'b111: result = a >> 1;                        // Logical shift right
        default: result = 8'b0;
    endcase
end

endmodule
```

### Truth Table (operation select)

| `sel` | Operation | `result` | `carry_out` |
|---|---|---|---|
| `000` | ADD | `a + b` | carry out of MSB (if applicable) |
| `001` | SUB | `a - b` | borrow flag |
| `010` | AND | `a & b` | 0 |
| `011` | OR | `a \| b` | 0 |
| `100` | XOR | `a ^ b` | 0 |
| `101` | NOT | `~a` (ignores `b`) | 0 |
| `110` | SHL | `a << 1` | 0 |
| `111` | SHR | `a >> 1` | 0 |

Example with `a = 8'b00000110` (6), `b = 8'b00000011` (3):

| `sel` | Operation | `result` |
|---|---|---|
| `000` | ADD | `00001001` (9) |
| `001` | SUB | `00000011` (3) |
| `010` | AND | `00000010` (2) |
| `011` | OR | `00000111` (7) |
| `100` | XOR | `00000101` (5) |
| `101` | NOT | `11111001` |
| `110` | SHL | `00001100` (12) |
| `111` | SHR | `00000011` (3) |

## 1. Project Setup

```bash
mkdir -p ~/vlsi/projects/alu8/src
cd ~/vlsi/projects/alu8
```

Save the RTL above as:

```bash
nano src/alu8.v
```

---

## 2. `config.json`

```bash
nano config.json
```

```json
{
    "DESIGN_NAME": "alu8",
    "VERILOG_FILES": [
        "dir::src/alu8.v"
    ],
    "CLOCK_PORT": null,
    "FP_SIZING": "absolute",
    "DIE_AREA": [0, 0, 300, 300],
    "CORE_AREA": [20, 20, 280, 280]
}
```

## 3. Enter the OpenLane Nix Shell

```bash
cd ~/vlsi/openlane2
nix-shell
```

---

## 4. Run the Flow

```bash
cd ~/vlsi/projects/alu8
openlane --design-dir . config.json
```

```bash
find ~/vlsi/projects/alu8 -name "*.gds"
```

## 6. Open in KLayout

```bash
klayout ~/vlsi/projects/alu8/runs/RUN_2026-07-12_20-47-xx/final/gds/alu8.gds
```

Inside KLayout:

| Key | Action |
|---|---|
| `F` | Fit layout to window |
| Mouse wheel | Zoom |
| Middle-click drag | Pan |

## 7. Open in Magic

```bash
magic -T ~/.volare/volare/sky130/versions/0fe599b2afb6708d281543108caf8310912f54af/sky130A/libs.tech/magic/sky130A.tech
```

Inside the **tkcon** window (Magic's own command console — not the Linux terminal):

```
gds read ~/vlsi/projects/alu8/runs/RUN_2026-07-12_20-47-xx/final/gds/alu8.gds
cellname list
load alu8
view
```

## Full Command Sequence (Copy-Paste Order)

```bash
# 1. Set up project
mkdir -p ~/vlsi/projects/alu8/src
cd ~/vlsi/projects/alu8
nano src/alu8.v        # paste RTL
nano config.json        # paste config

# 2. Enter OpenLane environment
cd ~/vlsi/openlane2
nix-shell

# 3. Run the flow
cd ~/vlsi/projects/alu8
openlane --design-dir . config.json

# 4. Find the GDS
find ~/vlsi/projects/alu8 -name "*.gds"

# 5. View in KLayout
klayout ~/vlsi/projects/alu8/runs/RUN_2026-07-12_20-47-xx/final/gds/alu8.gds

# 6. View in Magic
magic -T ~/.volare/volare/sky130/versions/0fe599b2afb6708d281543108caf8310912f54af/sky130A/libs.tech/magic/sky130A.tech
```

Then inside Magic's `tkcon`:

```
gds read ~/vlsi/projects/alu8/runs/RUN_2026-07-12_20-47-xx/final/gds/alu8.gds
cellname list
load alu8
view
```

## Note on the Wayland Warning

When launching `klayout` on this setup, the following is expected and harmless:

```
Info: Warning: Ignoring XDG_SESSION_TYPE=wayland on Gnome. Use QT_QPA_PLATFORM=wayland to run on Wayland anyway.
Gtk-Message: Failed to load module "appmenu-gtk-module"
Gtk-Message: Failed to load module "canberra-gtk-module"
```

KLayout falls back to X11/XWayland automatically — the layout still opens correctly. No action needed unless the window fails to appear at all.
